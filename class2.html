Flexbox

Display: flex

  • Flexbox is an incredibly powerful tool!
  • By using display: flex; we can use flexbox to position our items inside a flexible box.
  • Flexbox allows us to easily manupulate layout and display items where we want them on the page

How to Start

    
      

        
Item One

        
Item Two

        
Item Three

      

    
  
    
      .container {
        display: flex;
      }

Make sure to target the parent class!

Spacing Properties

    
      .container {
        display: flex;
        justify-content: space-around;
      }
    
  
    
      .container {
        display: flex;
        justify-content: center;
      }
      .container div {
        padding: 0 15px;
      }

Perfectly Centered!

    .container {
display: flex;
justify-content: center;
align-items: center;
height: 200px;
border: solid 1px black;
}

.container div {
padding: 0 15px;
}

We can even get our items to stack!

    
      .container {
        justify-content: center;
        flex-direction: column;
        align-items: center;
      }

Maybe we want our items reversed?

    
      .container {
        justify-content: center;
        flex-direction: row-reverse;
        align-items: center;
      }

What is a Hero?

  • Large banner image, prominently placed on a web page, generally in the front and center
  • First visual a visitor encounters on the site and its purpose is to present an overview of the site's most important content
  • Often consists of image and text, can be static or dynamic

Vendor Prefixes

Vendor Prefixes

HTML5 and CSS3 are still "new"!

This is great, but it means that not all browsers treat new CSS3 properties the same

Vendor Prefixes

Flags it as a work-in-progress

When finished, the browser should support the non-prefixed name

Vendor Prefixes

Each browser has their own:

  • Chrome:-webkit-
  • Firefox:-moz-
  • Safari:-webkit-
  • Opera:-o-

When do we use prefixes?

  • Most of what we are covering in class does not need a vendor prefix
  • Can I Use
  • Should I Prefix

Using Prefixes

.container {
          display: -webkit-box;  /* OLD - iOS 6-, Safari 3.1-6, BB7 */
          display: -ms-flexbox;  /* TWEENER - IE 10 */
          display: -webkit-flex; /* NEW - Safari 6.1+. iOS 7.1+, BB10 */
          display: flex;         /* NEW, Spec - Firefox, Chrome, Opera */
}

Order matters! The non-prefixed version should always go last.

NOTE:

While you should always use the vendor prefixes when necessary, today we're just going to use the non-prefixed property.

This is to save time during the code exercises

Background-size

Notice how the image is too large for the section? Let's fix that with a new property called background-size 

#hero {
background: url(../images/zebra.jpg) no-repeat;
color: #fafafa;
text-align: center;
height: 350px;
padding: 25px 0;
background-size: cover;
}

cover scales the image to the largest size such that both its width and its height can fit inside the content area.

Some other background image properties include:

  • background-position
  • background-repeat
  • background-size

3-column Content Area

Why 3 columns?

Because 2 columns is so 2013

Or, because it's a comfortable width for readability. And because 3 is a pleasing design construct.

3 Column: HTML

Our code in structure.html already looks like this:

<section id="main-content">
  <div class="container">
    <section class="column">
      <!-- Content goes here! -->
    </section>
    <!-- Repeat .column x3 -->
  </div>
</section>

3 Column: CSS

Now that we have our 3 columns, we want them to appear next to each other. We can do this with flexbox!

.container {
      display: flex;
      justify-content: space-between;
}

We used space-between, because it gives us perfectly spaced columns without doing math! Don't forget padding (or margin) to give the columns some space.
